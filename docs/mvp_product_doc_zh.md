# 日本超市价格比价工具
## MVP 产品开发文档（产品经理版）

- **版本**：V1.0
- **范围**：第一版 MVP
- **产品形态**：移动端优先网页应用

---

## 1. 项目背景
- 目前使用飞书多维表格记录日本超市商品价格，包括商品名称、价格、规格、店铺、图片、条形码等信息。
- 飞书多维表格适合早期验证，但移动端体验、图片加载、商品聚合和最低价展示不够适合长期使用。
- 需要开发一个独立的、移动端优先的价格记录与比价工具，替代飞书表格作为小范围使用的正式产品。

## 2. 产品目标

### 2.1 核心目标
帮助用户快速回答一个问题：**某个商品现在在哪个超市买最便宜**。这里的“最便宜”不仅看总价，也要看单位价格。

- ¥178 / 400g = ¥44.5 / 100g
- ¥348 / 800g = ¥43.5 / 100g

结论：第二个商品虽然总价更高，但单位价格更低。

### 2.2 MVP 阶段目标
1. 用户可以手动记录商品价格。
2. 用户可以上传商品图或价签图。
3. 系统可以自动计算单位价格。
4. 用户可以按商品查看不同店铺价格。
5. 用户可以快速判断哪个店铺单位价格最低。
6. 用户可以搜索商品，并按分类、店铺筛选。

### 2.3 非目标
- 不做 AI 拍照识别。
- 不做 OCR 自动识别价签。
- 不做原生 iOS / Android App。
- 不做复杂社区功能。
- 不做自动抓取超市线上价格。
- 不做价格提醒、积分、排行榜和评论体系。

## 3. 用户画像

| 项目 | 说明 |
| --- | --- |
| 初期用户 | 日本留学生、小范围朋友群体，约 1–5 人。 |
| 使用设备 | 手机为主，购物现场快速记录。 |
| 用户特征 | 价格敏感，经常在日本超市、药妆店、业务超市、Cosmos、Donki 等购物。 |
| 核心诉求 | 记录常买商品价格，比较同一商品在不同店铺的价格和单位价格。 |

## 4. 核心使用场景

| 场景 | 用户动作 | 系统结果 |
| --- | --- | --- |
| 在超市记录价格 | 打开应用，添加商品名、价格、规格、店铺和图片。 | 保存价格记录，自动计算单位价格。 |
| 购物前查询最低价 | 搜索商品，例如“片栗粉”。 | 展示当前最低单位价、对应店铺和最近记录。 |
| 比较大小包装 | 查看 400g 与 800g 两种规格。 | 显示每 100g 单价，帮助判断哪个更划算。 |
| 记录日用品 | 记录纸巾、洗洁精、洗衣液等。 | 支持每个、每包、每盒等数量型单位。 |

## 5. 产品形态
第一版本推荐采用移动端优先网页应用。用户通过链接访问，也可以添加到手机桌面，接近 App 使用体验。

- 不需要上架 App Store 或 Google Play。
- 不需要处理 iOS / Android 双端开发。
- 适合 1–5 人小规模使用。
- 开发和维护成本低。
- 后续可继续扩展为 PWA 或原生 App。

## 7. 页面需求

### 7.1 首页：商品价格列表
页面目标：让用户快速看到已记录商品，并判断哪些商品最近记录过、哪个价格更低。

| 展示字段 | 说明 |
| --- | --- |
| 商品图片 | 缩略图，帮助识别商品。 |
| 商品中文名 | 主要搜索字段。 |
| 商品日文名 | 日本商品原名。 |
| 店铺名称 | 价格来源店铺。 |
| 税后价格 | 实际支付价格。 |
| 规格 | 例如 400g、900ml、6个。 |
| 单位价格 | 例如 ¥44.5 / 100g。 |
| 记录日期 | 该价格的记录时间。 |
| 分类 | 食品、日用品等。 |

支持能力：
- 搜索商品。
- 按分类筛选。
- 按店铺筛选。
- 点击商品卡片进入详情页。
- 点击添加按钮进入添加页。

### 7.2 添加价格记录页
页面目标：让用户用最少步骤完成一条价格记录。

| 字段 | 是否必填 | 说明 |
| --- | --- | --- |
| 商品中文名 | 必填 | 例如：片栗粉。 |
| 商品日文名 | 可选 | 例如：片栗粉。 |
| 品牌 | 可选 | 例如：Topvalu、Nissin。 |
| 条形码 | 可选 | 后续用于识别同一商品。 |
| 店铺 | 必填 | 例如：Cosmos、业务超市。 |
| 分类 | 必填 | 食品、日用品等。 |
| 税后价格 | 必填 | 实际用于比价。 |
| 税前价格 | 可选 | 用于展示或补充。 |
| 税率 | 可选 | 0%、8%、10%。 |
| 规格数值 | 必填 | 例如：400。 |
| 单位 | 必填 | g、kg、ml、L、个、袋、包等。 |
| 图片 | 可选 | 商品图或价签图。 |
| 备注 | 可选 | 促销、会员价、限时价格。 |
| 记录日期 | 必填 | 默认当天。 |

交互规则：
- 自动计算单位价格。
- 保存商品信息。
- 保存价格记录。
- 如果商品已存在，则关联到已有商品。
- 如果商品不存在，则创建新商品。
- 保存后返回首页或商品详情页。

### 7.3 商品详情页
页面目标：展示某个商品在不同店铺、不同时间的价格记录，并突出最低单位价。

| 模块 | 内容 |
| --- | --- |
| 商品基础信息 | 中文名、日文名、品牌、分类、条形码、默认图片。 |
| 价格概览 | 当前最低单位价、当前最低总价、最近更新时间、记录数量。 |
| 价格记录列表 | 店铺、税后价格、规格、单位价格、日期、备注。 |
| 交互能力 | 按单位价格排序、按日期排序、点击图片查看大图。 |

### 7.4 店铺管理页
页面目标：管理常用超市和店铺。MVP 第一版只需要查看店铺列表和新增店铺。

| 字段 | 说明 |
| --- | --- |
| 店铺名称 | 例如：中央 cosmos。 |
| 连锁品牌 | 例如：Cosmos。 |
| 位置 | 例如：中央、天神、博多。 |
| 备注 | 可选。 |

### 7.5 我的页面
MVP 第一版可以非常简单：当前用户昵称、退出登录、已记录商品数、已记录价格数、已记录店铺数。

## 9. 商品识别与合并规则
- 条形码相同：默认认为是同一个商品。
- 没有条形码：根据中文名、日文名、品牌、分类判断是否可能相同。
- MVP 不建议自动强制合并，只做“发现可能相同商品，是否使用已有商品”的提示。
- 同类商品不等于同款商品，例如不同品牌的方便面不应强行合并。

## 10. 单位价格计算规则

| 类型 | 适用单位 | 统一展示 | 示例 |
| --- | --- | --- | --- |
| 重量类 | g、kg | 每 100g 价格 | 400g ¥178 = ¥44.5 / 100g。 |
| 容量类 | ml、L | 每 100ml 价格 | 900ml ¥298 = ¥33.1 / 100ml。 |
| 数量类 | 个、袋、包、盒、枚、本、瓶、罐、卷 | 每 1 个单位价格 | 6个 ¥198 = ¥33 / 个。 |

价格口径：MVP 第一版优先使用税后价格，因为用户实际支付的是税后价格，且日本商品可能存在 8% 和 10% 税率差异。

## 11. 图片需求
- 支持上传商品图、价签图、货架图。
- 首页显示缩略图，详情页支持查看较大图片。
- 不上传原始大图，上传前压缩。
- 列表页只加载缩略图，图片懒加载。
- 图片加载失败时显示默认占位图。
- 每条价格记录可以有一张图片，商品默认图取最近一次上传的图片。

## 12. 搜索、筛选与排序

| 能力 | 范围 |
| --- | --- |
| 搜索 | 商品中文名、商品日文名、品牌、条形码。 |
| 筛选 | 分类筛选、店铺筛选。 |
| 排序 | 最近记录优先、单位价格从低到高、总价从低到高。 |

如果开发资源有限，第一版只做“最近记录优先”。

## 13. 权限与用户规则
- MVP 建议使用邮箱登录。
- 所有登录用户都可以查看全部价格数据。
- 所有登录用户都可以新增商品和价格记录。
- 编辑权限建议为“谁创建，谁编辑”。如果开发复杂，第一版可先不做编辑。
- MVP 不建议开放删除，后续可增加软删除能力，避免误删数据。

## 15. 关键用户流程

### 15.1 新增价格记录流程
打开应用 → 点击添加 → 输入商品名 → 选择店铺和分类 → 输入税后价格、规格和单位 → 系统实时显示单位价格 → 上传图片 → 点击保存 → 返回首页或商品详情页

### 15.2 查询最低价流程
打开应用 → 搜索商品名 → 进入商品详情 → 查看不同店铺价格 → 系统标记最低单位价格 → 用户决定去哪家店购买

## 16. 交互原则
- 添加记录要快：购物现场不能让表单过长，必填项优先。
- 比价结果要直观：商品详情页直接显示最低单位价、对应店铺和对应日期。
- 图片不能拖慢主流程：图片上传失败时，仍应允许保存文字记录。
- 避免误合并商品：除非条形码相同，否则不自动强制合并。

## 17. 首页展示逻辑

| 方案 | 优点 | 缺点 | 建议 |
| --- | --- | --- | --- |
| 方案 A：展示所有价格记录 | 实现简单，接近飞书表格逻辑。 | 同一商品可能出现多次，不够像比价工具。 | 适合最早期版本。 |
| 方案 B：按商品聚合展示 | 更像真正的比价工具，可直接展示最低价。 | 实现略复杂，需要聚合逻辑。 | 如果开发能力允许，第一版直接做。 |

推荐 MVP 首页卡片展示：商品图片、商品名、当前最低单位价、对应店铺、最近记录价格和日期。

## 18. 数据质量规则
- 价格必须大于 0。
- 规格必须大于 0。
- 单位必须选择，不能只填数字。
- 店铺必须选择，如果没有店铺，应先新增。
- 商品名必须填写，至少填写中文名或日文名之一。

## 19. 异常状态

| 状态 | 提示文案 |
| --- | --- |
| 没有数据 | 还没有价格记录，点击添加第一条商品价格。 |
| 搜索无结果 | 没有找到相关商品，可以新增一条价格记录。 |
| 图片加载失败 | 显示默认图片占位符。 |
| 网络失败 | 网络异常，请稍后重试。 |
| 保存失败 | 保存失败，请检查网络后重试。 |

1. 用户可以登录。
2. 用户可以新增商品价格记录。
3. 用户可以选择店铺和分类。
4. 用户可以输入价格、规格和单位。
5. 系统可以自动计算单位价格。
6. 用户可以上传图片。
7. 首页可以展示价格记录。
8. 用户可以搜索商品。
9. 用户可以筛选分类和店铺。
10. 用户可以进入商品详情页。
11. 商品详情页可以展示不同店铺价格。
12. 系统可以标记最低单位价格。
13. 手机端浏览体验正常。
14. 5 名左右用户同时使用没有明显卡顿。
15. 数据保存在云端，不依赖本地设备。

| 问题 | 建议 |

| 价格口径 | 税后价格为核心，税前价格可选填。 |

## 24. MVP 一句话定义
一个移动端优先的日本超市商品价格记录工具，用户可以手动记录商品在不同店铺的价格和规格，系统自动换算单位价格，并帮助用户判断哪里最便宜。
