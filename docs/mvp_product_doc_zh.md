# 日本超市价格比价工具
## 当前产品文档

- **当前形态**：移动端优先网页应用。
- **移动端适配**：所有页面统一禁止整页双指缩放，避免误触导致应用比例错乱；图片裁剪区域内仍支持双指缩放。
- **部署方式**：Cloudflare Pages + D1 + R2。
- **使用范围**：个人到小范围朋友使用，优先免费方案。

## 1. 项目背景

原始数据来自飞书多维表格。飞书适合早期记录，但在手机端录入、图片管理、商品聚合、最低价展示和权限控制上不适合长期使用。因此当前项目目标是替代飞书，成为小范围使用的正式价格记录和比价工具。

## 2. 产品目标

核心问题：**这个商品现在在哪个店买更便宜。**

比价口径：

- 税后价格用于实际支付和走势展示。
- 单位价格用于不同规格比价。
- `g/kg` 统一为 `/100g`。
- `ml/L` 统一为 `/100ml`。
- `个/pack` 按对应数量单位。

## 3. 当前页面

| 页面 | 当前作用 |
| --- | --- |
| 首页 | 新手教程、常用入口、优惠信息；不显示开发者专用入口。 |
| 清单 | 商品聚合列表、搜索、最低价标记、商品详情入口。 |
| 录入 | 新增商品价格记录，支持扫码、拍照/相册、快速填入、重复商品提示。 |
| 店铺 | 店铺列表公开查看，登录后新增/编辑/删除店铺。 |
| 我的 | 贡献统计、批量导入、用户名、绑定邮箱、反馈入口、开发者模式；管理员打开开发者模式后显示开发者工具入口。 |
| 日语读音标注 | 开发者/管理员专用；输入日语句子，标注汉字读音，本地保存生词和熟知词。 |

## 4. 商品清单

清单按商品聚合展示，不再按每条价格记录平铺。

展示重点：

- 商品图片。
- 中文名 / 日文名。
- 最新记录的税后价格、店铺、日期。
- 单位价格。
- 当前最低价标记。
- 限时优惠标记。

搜索范围：

- 全部。
- 只搜名称。
- 只搜店铺。

空结果会显示引导，点击可进入录入页。

批量修改：

- 清单页右上角提供批量修改入口，也可长按商品卡片进入批量选择。
- 支持单选、多选、全选当前展示的商品记录。
- 批量操作会修改每个商品当前展示的最新价格记录。
- 当前可批量修改购买店铺、记录日期、限时优惠和备注，提交前需要二次确认。

## 5. 商品详情

详情页用于查看一个商品的当前记录和历史变化。

当前包含：

- 紧凑商品图片。
- 中文名和日文名。
- 税后总价。
- 对比单价（按税后价计算）。
- 规格。
- 记录时间。
- 创建者和最后修改者。
- 各店现价排行。
- 历史记录列表。
- 税后价格走势。
- 头部图标快捷新增价格记录。
- 头部图标快捷修正记录。
- 删除当前记录。

修正记录时可编辑中文名、日文名、价格、规格、店铺和限时优惠；记录日期由系统自动写入当天，不在表单中手动填写。

价格走势使用税后价格，不使用单位价格走势。不同店铺之间不连线，只显示散点；同一店铺跨日期记录才用折线连接。

清单卡片优先显示中文名；同时存在中文名和日文名时分两行显示，过长名称用省略号截断。

## 6. 录入页

必填规则：

- 商品中文名或日文名至少填一个。
- 购买店铺必填。
- 税前价格必填。
- 税率必选。
- 规格数值可留空；如果当前单位有默认规格，则提交时自动补默认值。
- 规格单位必选。
- 记录日期由系统自动写入当天，不在录入表单中显示。

当前交互：

- 条码通过右上角扫码，不手动输入为主。
- 品牌和分类不是核心字段，分类暂不作为主要录入项。
- 购买店铺使用下拉框选择，并默认带入用户上次选择的店铺。
- 快速填入识别到的价格默认按税前价格带入，不直接覆盖税后价格。
- 税后价自动计算，灰底显示，提示用户不用手动输入；自动计算统一使用 `税前价 × (1 + 税率)`，并用 `Math.round` 四舍五入到整数日元。
- Cosmos 选择 0% 税率时可自动匹配；其他超市默认 8%。
- 规格数值可打开计算器，适合 `100g * 3` 这类组合。
- 规格单位 `g / ml / 个 / pack` 使用单行分段按钮显示。
- 规格数量输入框会随单位显示默认提示：`g = 1000`、`ml = 1000`、`个 = 1`、`pack = 1`；用户手填优先，留空时提交自动补默认值。
- 限时优惠和优惠截止日期同一行显示；无优惠时截止日期保持可见但置灰不可选。
- 图片可拍照或从相册选择，先自由比例裁剪再由前端压缩后提交，最多保留 4 张；裁剪支持拖动、拖边/角调整裁剪框、滑杆缩放、双指缩放和上下/左右翻转；详情页点击图片可查看大图，编辑/录入页已有图片时会先选择查看大图或更换图片。
## 7. 商品识别与重复处理

规则：

- 条码相同：视为同一个商品，不能重复创建新商品，只追加价格记录。
- 没有条码：中文名相同允许创建不同商品，因为可能只是泛称相同。
- 录入页发现可能相似商品时，提示用户查看已有商品或继续录入。

## 8. 最低价规则

当前存在两类逻辑：

- 最低价：按商品名称关键词包含关系，在相关商品中找最低单位价。
- 同产品最低价：条码相同则按条码；无条码时按中文名。当前暂不在外层强展示，后续需要时再打开。

详情页不重复堆太多最低价说明，避免信息过载。

## 9. 店铺页

当前店铺页：

- 未登录可查看。
- 点击店铺跳转清单，并筛选该店铺商品。
- 登录后可新增店铺。
- 编辑/删除不放在卡片最外层，使用长按或左滑显示操作。
- 新增店铺时用户只需输入名称；可搜索东广岛市内相关超市结果，再二次确认。

店铺名称显示规则偏向中国用户可读性，例如：

- `FRESTA フレスタ（西条店）`
- `小youme（广岛大学附近）`
- `大youme（東広島）`
- `youme mall ゆめモール（西条）`

## 10. 我的页面和反馈

未登录点击“我的”会显示登录弹窗，可返回或选择登录。

登录后显示：

- 用户名。
- 绑定邮箱。
- 自己添加的价格记录数。
- 自己创建的商品数。
- 自己创建的店铺数。
- 修改次数。

记录与模板区域：

- 可下载 CSV 模板；模板包含中文名、日文名、条码、店铺、价格、规格、日期、限时优惠、优惠截止日期和备注。
- 上传 CSV 后进入导入确认列表。
- 导入确认列表支持全选/单选、字段手动修正、逐行校验；只有确认导入后才会批量写入价格记录。
- 浏览器端暂不直接解析 `.xlsx/.xls`，Excel 表格需要先另存为 CSV 再上传。

首次登录要求设置用户名。

普通用户可提交反馈。管理员打开开发者模式后才能查看所有反馈。

开发者模式额外显示“数据整理”：

- 可一键导出所有价格记录为 Excel 兼容 CSV。
- 导出的开发者 CSV 默认包含 `recordId`、`productId`、商品名、条码、单列图片 URL、店铺、税前价、税率(%)、规格、日期、优惠和备注。
- 开发者 CSV 只给用户展示税前价，不展示税后价；`storeId` 不再导出给用户。
- CSV 可在表格软件中直接修改商品名、条码、店铺、价格、规格、日期、限时优惠和备注。
- 条码导出时按文本写出，避免 Excel/Numbers 打开时被转科学记数法或吞前导零。
- 上传时会跳过整行空白的数据行，不会按默认值误判为新增记录。
- 上传时会自动把 `YYYY/M/D`、`YYYY-M-D` 和 Excel 日期序列号规范为 `YYYY-MM-DD`。
- 上传校验失败时会提示具体字段值和该行关键字段，便于定位表格问题。
- 上传修改后的 CSV 时，带 `recordId` 的行会更新原记录，空 `recordId` 的行会新增价格记录。
- `recordId` 是价格记录唯一编号；同一家店出现多条记录是正常的，因此导出里不再展示 `storeId`。
- 批量更新前需要二次确认，建议先导出备份。

## 11. 权限规则

- 清单、店铺列表公开查看。
- 录入、个人数据、写入接口需要登录。
- 所有登录用户可以新增商品和价格记录。
- 所有登录用户可以修改别人的价格记录。
- 普通用户只能删除自己创建的数据。
- 管理员可以删除所有人数据。
- 普通用户只能撤回自己的修改。
- 管理员可以撤回他人修改。

## 12. 数据存储

线上：

- 商品、店铺、价格记录、用户资料、反馈：Cloudflare D1。
- 图片：Cloudflare R2。

本地：

- `data/app.json`。

日语读音标注页：

- 仅开发者/管理员可见、可访问。
- 当前只使用浏览器 `localStorage`。
- 不写入 D1。
- 不跨设备同步。

## 13. 当前非目标

- 不做付费 AI。
- 不做 OCR 自动识别价签。
- 不做日亚官方 API 自动抓图。
- 不做价格提醒。
- 不做复杂社区。
- 不做 App Store / Google Play 原生应用。
